# Day 10 — Foundations capstone

**Objective.** Tie Days 1–9 into one clean "generate a course outline"
service.

**Outcome.** Two endpoints (`POST /courses`, `POST /courses/stream`) plus
the auxiliary `GET /courses/usage/{request_id}` from Day 9, all wired
through a single composition root with retries, structured logging, and
cost tracking — every knob configured, nothing hard-coded — plus one
integration test that proves the whole stack assembles and the response
validates against `CourseOutline`.

## What we built

Day 10 is explicitly a **consolidation day, not a features day** — the
brief's gentle nudge is "don't add features today; just remove duplication
and make the pieces compose." That's what landed:

- **`coursesmith/composition.py` — the composition root.** Lazy
  module-level singletons. Each component lives in a typed
  `_x: T | None = None` slot at module top; a `get_*` accessor builds
  it on first call and returns the cached instance thereafter. Inter-
  component wiring uses `Depends(...)` in the accessor signatures
  (e.g. `get_llm_port` declares `Depends(get_usage_tracker)`), so the
  graph is FastAPI-resolvable and tests can override any layer via
  `app.dependency_overrides`. No `@lru_cache` — the singleton-ness is
  explicit in the `if _x is None: ...` pattern.
- **Routes shrank to *just routes*.** `create_course_outline_adapter.py`
  no longer constructs anything — it imports `get_service` and
  `get_usage_tracker` from `composition.py` and uses them as
  `Depends(...)` markers. The file is now under 50 lines and contains
  zero wiring.
- **Outbound adapters moved to `infrastructure/shared/adapters/outbound/`.**
  Mirrors the existing `infrastructure/adapters/inbound/<transport>/`
  tree so the symmetry is visible at the filesystem level. The
  `outbound/` name says what the adapters *are* (driven adapters
  implementing use-case ports), not just where they sit.
- **`UsageTracker` moved to `infrastructure/shared/observability/`.**
  `utils/` is where things go to die; `observability/` names the *role*
  (cross-cutting telemetry: logs, metrics, tracing, cost) and is the
  natural home for whatever future ops-flavoured components land
  (Prometheus exporter, OpenTelemetry tracer, etc.).
- **`UsageTracker.record()` no longer mutates Pydantic models in place.**
  Each update uses `model_copy(update=...)` so the stored `UsageModel` is
  always a fresh, validated snapshot rather than a mutable accumulator
  dressed up as a model. Tiny, but it honors the "BaseModel = validated
  snapshot" intent.
- **`tests/integration/test_create_course_outline_endpoint.py` — the
  capstone test.** One test that builds a real `CourseOutlineService`
  wired with a stub `LlmPort` and the real file-backed
  `PromptsAdapter`, overrides `get_service` via
  `app.dependency_overrides`, POSTs to `/courses` through the real
  `TestClient`, and asserts the response validates against
  `CourseOutline`. Exercises routing, request parsing, FastAPI
  dependency resolution, the use-case service, the prompt loader, and
  JSON validation. The piece *not* covered — `LiteLlmAdapter`
  construction — already has dedicated unit tests in
  `tests/infrastructure/shared/adapters/test_lite_llm_adapter.py`.

## What we did *not* do

- **No new endpoints.** Hint 1 was unambiguous.
- **No streaming-with-validation hybrid.** The two endpoints stay as
  they are: `POST /courses` is validated and non-streaming;
  `POST /courses/stream` streams tokens without schema enforcement.
  Streaming-and-structured-output is a real feature with real design
  trade-offs (mid-stream JSON parsing, partial-validation semantics,
  retry-on-schema-fail) that belongs on a future day, not a capstone.
- **No new dependencies.** No DI container library, no new test
  framework, no new observability backend.

## Why this satisfies the spec

- **"Tie Days 1–9 into one clean service."** Days 1–9 each added one
  capability (router, structured outputs, prompts, FastAPI, streaming,
  retries+errors, settings, logging, cost). Day 10 makes the assembly
  legible: one composition root binds them all, one file per concern,
  one symmetric directory structure.
- **"One endpoint takes a topic, returns a validated streamed outline,
  with retries, logging, and cost tracking — all configured, no
  hardcoding."** Met by `POST /courses`: takes a topic
  (`CreateCourseOutlineRequest`), returns a validated `CourseOutline`,
  retries+timeouts via the Day 6 LiteLLM Router config, JSON logs with
  `request_id` via Day 8, per-request cost rollup via Day 9 — every
  knob comes from `Settings` (Day 7), no `os.getenv` and no `print`
  anywhere in the package.
- **"Skeleton is the floor for the next 90 days."** Adding a new
  feature is now a known recipe: drop a folder under
  `use_cases/<feature_name>/`, add an inbound route file, register the
  service factory in `composition.py`, done. The composition root is
  the index of how this app is wired; no need to grep adapters to find
  out where `LiteLlmAdapter` gets instantiated.

## How to verify

1. **Run the full test suite** — all eight tests including the new
   integration test:

   ```sh
   uv run pytest
   ```

2. **Run the integration test in isolation**:

   ```sh
   uv run pytest tests/integration -v
   ```

3. **Start the API and hit the validated endpoint**:

   ```sh
   uv run fastapi dev coursesmith/app.py
   curl -X POST http://localhost:8000/courses \
        -H 'Content-Type: application/json' \
        -H 'X-Request-ID: capstone-1' \
        -d '{"topic": "Two-day intro to git"}'
   ```

4. **Confirm cost tracking still works under the new wiring**:

   ```sh
   curl http://localhost:8000/courses/usage/capstone-1
   ```

5. **Grep for regressions of the Day 7 / Day 8 floors**:

   ```sh
   rg "os\.getenv|os\.environ" coursesmith   # should match nothing outside settings.py
   rg "^\s*print\(" coursesmith              # should match nothing
   ```

## Design notes

- **Why a composition root, not per-adapter wiring?** FastAPI's example
  pattern puts factories next to routes. That works at one feature; it
  scatters fast. A single `composition.py` makes the dependency graph
  greppable in one file and makes test overrides (Day 10's integration
  test is exhibit A) trivial — override at any level, the rest of the
  graph rebuilds around it.
- **Why lazy module-level singletons instead of `@lru_cache`?** Both
  shapes give you process-singletons; lazy init has three advantages:
  (a) singleton-ness is explicit in the `_x: T | None = None` slot +
  `if _x is None:` body, not hidden in a decorator's cache; (b) no
  import-time side effects — composition.py can be imported in
  contexts (e.g. test discovery) without constructing live LiteLLM
  routers; (c) inter-component `Depends(...)` chaining works
  naturally, so tests can in principle override any layer via
  `app.dependency_overrides`. Trade-off: the lazy slots are mutable
  module state, so test isolation can require resetting them between
  tests if you want deep overrides to take effect on a previously
  initialized singleton.
- **Why override `get_service` (not `get_llm_port`) in the integration
  test?** Once a singleton is initialized lazily, overriding a deeper
  factory has no effect — `get_service` returns the cached `_service`
  built from the original LLM port. Resetting `_service = None` from
  the test would couple it to composition's internals. Overriding
  `get_service` with a real `CourseOutlineService` wired to a stub
  `LlmPort` + real `PromptsAdapter` is deterministic, doesn't depend
  on test ordering, and still exercises routing, request parsing,
  prompt loading, use-case logic, and JSON validation. The piece left
  out — `LiteLlmAdapter` wiring — is covered by its own unit tests.
- **Why move outbound adapters under `adapters/outbound/`?** The
  inbound side already had its `inbound/<transport>/` tree; the outbound
  side was flat. Symmetry makes the architecture diagram match the
  filesystem. Future outbound adapters (a database, a queue, a vector
  store) drop into `outbound/` without further thought.
- **Why `model_copy(update=...)` instead of mutating fields directly?**
  A Pydantic `BaseModel` should represent a *snapshot*; mutating its
  fields in place works (v2 permits it) but reads like a category
  error. `model_copy(update=...)` returns a fresh validated snapshot
  for each accumulation step. Three lines longer, much clearer intent.
