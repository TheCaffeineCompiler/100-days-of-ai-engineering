# Day 4 — First FastAPI endpoint

**Objective.** Expose course generation over HTTP.

**Outcome.** `POST /courses` with `{"topic": "..."}` returns a JSON
`CourseOutline`; the endpoint is visible in `/docs`.

## What we built

- **`coursesmith/app.py`** — the FastAPI composition root. Constructs the
  `FastAPI()` instance and mounts the courses router. Replaces the previous
  CLI entry point (`hello.py` removed).
- **Inbound-adapter layer.** New tree under
  `coursesmith/infrastructure/adapters/inbound/rest/` with
  `create_course_outline_adapter.py`. The hexagonal layout now has an explicit
  *inbound* vs *outbound* split:
  - `infrastructure/adapters/inbound/<transport>/` — handlers that translate
    incoming requests into use-case calls (today: REST; later: maybe gRPC,
    background workers, …),
  - `infrastructure/shared/adapters/` — outbound adapters that implement
    use-case ports (today: file-backed prompts).
- **`POST /courses` endpoint** wired through FastAPI:
  - Request body model `CreateCourseOutlineRequest { topic: str }`.
  - Response model `CourseOutline` (`response_model=` makes it visible in the
    auto-generated OpenAPI schema and Swagger UI at `/docs`).
  - Async handler that `await`s `service.create(topic=...)`.
- **`CourseOutlineService` made async end-to-end.** Switched from
  `litellm.completion(...)` to `await litellm.acompletion(...)` so the LLM
  call no longer blocks the event loop. The HTTP layer can now serve
  concurrent requests.
- **FastAPI dependency injection.** Service construction lives in
  `get_service()` (decorated with `@lru_cache` so it builds once per process)
  and is injected into the route via `Depends(get_service)`. Tests can swap
  the real service for a fake via `app.dependency_overrides[get_service] =
  ...` without touching env vars or making real LLM calls.
- **`.env` loading via `SettingsConfigDict`.** Now that `hello.py` is gone and
  there's no `load_dotenv()` call, `Settings` reads `.env` natively:
  `model_config = SettingsConfigDict(env_file=".env", extra="ignore")`. Works
  for `uvicorn`/`fastapi dev`, pytest, anywhere else `Settings` is
  instantiated.
- **`topic` rename pushed through the stack.** The word `topic` now flows
  unbroken: HTTP body field → service parameter → prompt template variable.
  `title` only exists on the *output* (`CourseOutline.title`), which is the
  course title the LLM produces.
- **First endpoint test** at
  `tests/infrastructure/adapters/inbound/rest/test_create_course_outline_adapter.py`.
  Uses `fastapi.testclient.TestClient` plus
  `app.dependency_overrides[get_service]` with a `SimpleNamespace(create=AsyncMock(...))`
  fake. Proves the contract — `{"topic": "..."}` in, JSON `CourseOutline`
  out — with zero LLM calls.
- **Ruff bugbear config.** Added
  `[tool.ruff.lint.flake8-bugbear] extend-immutable-calls = ["fastapi.Depends",
  "fastapi.Query", ...]` so `B008` ("function call in argument defaults")
  stops flagging FastAPI's idiomatic DI markers. Future endpoints can use
  `Query()`, `Body()`, `Header()`, etc. as defaults without lint pain.
- **`fastapi[standard]`** added as a project dependency (pulls in
  `uvicorn`, `httptools`, the CLI, etc.).

## Why this satisfies the spec

- **"Expose course generation over HTTP."** `POST /courses` is mounted on a
  FastAPI app served by uvicorn.
- **"`POST /courses` with `{"topic": "..."}` returns a JSON CourseOutline."**
  The request body's only field is `topic`. The handler returns a validated
  `CourseOutline` Pydantic instance, which FastAPI serializes to JSON.
- **"Visible in `/docs`."** `response_model=CourseOutline` and the explicit
  request model make the endpoint show up in the auto-generated Swagger UI
  at `http://localhost:8000/docs`, with editable example payloads.

## How to verify

1. **Start the server**:

   ```sh
   uv run fastapi dev coursesmith/app.py
   ```

   (Or `uv run uvicorn coursesmith.app:app --reload` if you prefer uvicorn
   directly.)

2. **Open Swagger UI**: <http://localhost:8000/docs>. The `POST /courses`
   endpoint should be listed under the `courses` tag, with a `topic` body
   field and a `CourseOutline` response schema.

3. **Hit the endpoint**:

   ```sh
   curl -X POST http://localhost:8000/courses \
        -H 'Content-Type: application/json' \
        -d '{"topic": "AI engineering for backend developers"}'
   ```

   Returns a JSON `CourseOutline` (`{"title": "...", "day_items": [...]}`).

4. **Run the tests**:

   ```sh
   uv run pytest
   ```

   Two tests should pass — the Day 3 prompts adapter contract test and the
   new endpoint test.
