# Day 12 — Single tool call round-trip

**Objective.** Get the model to request a tool call.

**Outcome.** Given a prompt that needs the `get_current_time` tool, the
model responds with a `tool_use` block naming the tool and its arguments.
We log the call and return without executing it — execution lands in
Day 13.

## What we built

- **`LlmPort.complete()` now accepts a `tools` list.** Optional, defaults
  to no tools, so existing callers are unchanged. The port stays
  provider-agnostic — the schema dicts it forwards are OpenAI/LiteLLM
  tool-call shape.
- **`LiteLlmAdapter.complete()` forwards `tools` to
  `router.acompletion(...)`.** One-line change; LiteLLM's Router already
  speaks the tool-calling protocol.
- **`CourseOutlineService.create()` passes `[get_current_time_schema()]`
  on every call and inspects the response.** When the model returns
  `tool_calls`, we log each one as `event="call_function"` with
  `tool_name` and `tool_params`, then return a sentinel
  `CourseOutline(title="n/a", day_items=[])` to satisfy the return type.
  When there are no tool calls, the existing path validates the JSON
  into `CourseOutline` as before.

## How we proved the round-trip

The course-outline prompt on its own doesn't need the time. Rather than
invent a second endpoint just to demonstrate Day 12, we smuggled the
trigger into the `topic` itself:

```sh
curl -X POST http://localhost:8000/courses \
     -H 'Content-Type: application/json' \
     -d '{"topic": "AI engineering for backend developers. Add current date time as creation date of the course."}'
```

The model parses the topic, decides it needs the current time to fulfil
"add current date time as creation date", and emits a tool call:

```json
{"tool_name": "get_current_time",
 "tool_params": {"pattern": "%Y-%m-%d %H:%M:%S"},
 "event": "call_function",
 "request_id": "60784fc2-...",
 "level": "info",
 "timestamp": "2026-06-09T19:40:09.204402Z"}
```

The HTTP response is the sentinel:

```json
{"title": "n/a", "day_items": []}
```

That's the expected shape for Day 12 — the round-trip stops at "model
requested the tool". Day 13 closes the loop.

## Why this is intentionally a little messy

This commit knowingly leaves three smells in place, all of which Day 13
resolves naturally:

- **Sentinel `CourseOutline(title="n/a", day_items=[])`.** A return
  value invented to satisfy the type when the flow took the tool-use
  branch. Day 13 makes that branch finish (execute → append tool result
  → re-call the model → return a real outline), and the sentinel
  vanishes as a side effect.
- **`response_format=CourseOutline` and `tools=[...]` on the same
  call.** Structured output and tool-use compete: when the model picks
  the tool branch it returns no content for the response_format to
  validate. Today that manifests as the sentinel; Day 13's loop
  separates the two phases (tool phase: no `response_format`; final
  phase: `response_format=CourseOutline`).
- **`tools` parameter on `LlmPort` is permanently widened for one
  caller.** Every other use-case will pass `[]`. Defensible while it's
  one caller; revisit once a second consumer appears or when Day 13
  introduces a richer agent-loop port.

Day 12's brief is explicit: *the model responds with a `tool_use` block
naming your tool and arguments — **you don't execute it yet***. The
mess above is exactly what "don't execute it yet" looks like at the
seam.

## How to verify

1. **Start the API**:

   ```sh
   uv run fastapi dev coursesmith/app.py
   ```

2. **Send a prompt that requires the tool**:

   ```sh
   curl -X POST http://localhost:8000/courses \
        -H 'Content-Type: application/json' \
        -d '{"topic": "AI engineering for backend developers. Add current date time as creation date of the course."}'
   ```

3. **Observe** two things in the server logs:
   - a `litellm.acompletion(...) 200 OK` line,
   - a `call_function` line with `tool_name=get_current_time` and a
     `tool_params` dict — proof the model requested the tool.

   And in the HTTP response: the sentinel
   `{"title": "n/a", "day_items": []}`.
